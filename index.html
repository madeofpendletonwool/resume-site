Report Site

Previously, we'd have quite a few people manually assess systems at client sites for general issues, license expiries, and security problems. We'd then make individual work documents reporting on each device. I saw an oppurtunity there, and made it a goal of mine to automate as much of that process as possible. So I created our reporting site. It begins with a Jenkins job for the assessment of choice and then runs through quite the gambit of processes. It begins by creating a container instance to run an ansible job on the client site of choice. Ansible uses kerberos to authenticate with whatever computers it needs to or potentially ssh if it's a network device. Ansible will then run infomration gathering commands on the device in question, pull that data back to the server it's running on, and then remove any files it created when running. From there, Jenkins will run a php script on either a csv or txt file that it now has to convert that information into an html document formatted in the way seen above. From there, it scps the html document to the reports webserver in a specific location defined by variables and then send a clickable link to the assessment to the person who needs to recieve it in an email. This entirely prevents any human interaction at all if we scedule the job to run. I built this system to be adaptable as possible as well, so with minor changes in the commands we run this can gather information from Active Directory, to Excahnge servers, to Backup systems, to network devices like switches and firewalls. All it needs is a txt or csv file.

Automation, Web Design, PHP, Ansible, Github Actions, Python

Cecil

Cecil is my answer to health checks/monitoring of client infrastructure. The statically generated reports site builds some nice reports for clients to utilize however, we were missing a platform that can monitor devices, infrastructure, and network health. Cecil is a dockerized web application that gets deployed to client sites to a central container server that spins up a flutter web app. This application can be accessed from the client network at the ip of the container server and requires authentication to access using the internal company github for API access tokens. Once logged in you can setup 'modules'. Cecil is built as agnostically as possible and simply provides a framework in which further modules can be easily integrated on top of it to run additional monitors/alerts. These modules are simply python scripts that retrieve information. That information is then sent back to cecil where it forwards over https protocol using a load balancer to a central ntfy server. The ntfy server organizes the data and emails a ticket in to either alert, or provide a report.

Monitoring, Alerting, Flutter, Dart, Python

CAMM - Collin's Automated Machine Monitor

This is a fun little project I took up to handle the automation of a seemingly simple, yet time consuming issue we ran into with many clients. The scenario is this, we would onboard a client, get our tools ready (RMM, antivirus, etc.), and then have no way of knowing every the hostnames of every computer to deploy computers to. CAMM, was built to overcome this issue. CAMM is a python program I built that is containerized and can be deployed to a client site to monitor the network for any computers it finds on the network. It scans every so often and will add new computers it finds to a list. If after 60 days a computer is not seen it will be removed from the list. This allows us to keep an up to date list that we can automatically feed into an ansible hosts file for example to deploy tools too.

Monitoring, Pipeline Design, Automation
